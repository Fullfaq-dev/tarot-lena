# ADR-001: Продуктовая модель бота «Лея»

**Статус:** Принято (проектирование)  
**Дата:** 2026-07-06

## Контекст

Текущий Arcana AI построен вокруг баланса, подписок Plus/Premium и pay-per-token чата. ТЗ на бота «Лея» описывает другую модель: фиксированные разовые продукты, мини→полная воронка, комбо-пакеты и две подписки.

## Решение

1. **Заменить** tier-based подписки (`free/plus/premium`) на **product entitlements**:
   - разовые покупки с флагом `mini_used` / `full_unlocked`;
   - подписки `love_plus` и `vip` с `expires_at`;
   - комбо-пакет как набор из 3 entitlements.

2. **Сохранить** Platega как платёжный провайдер, но с новыми `payment.purpose` значениями (`product_*`, `subscription_*`, `combo_*`).

3. **Не использовать** `balance_rub` для покупки продуктов в MVP — только прямая оплата через Platega. Баланс можно оставить для будущих реферальных начислений.

4. **Паттерн мини→полная** реализовать единым `ProductService`:
   - `generate_mini(product_id, user)` — бесплатно, один раз на продукт (или по правилам воронки);
   - `generate_full(product_id, user)` — только при наличии entitlement или после оплаты.

## Последствия

- Потребуется миграция БД и рефакторинг `BillingService`.
- Admin-панель: новые метрики по продуктам вместо токенов.
- Старые handlers чата/vision/zen удаляются или скрываются.

## Альтернативы (отклонены)

- **Оставить баланс** — усложняет UX («пополни 550₽» вместо «купи Любовь»).
- **Один VIP без разовых** — противоречит ТЗ и снижает конверсию на входе.
